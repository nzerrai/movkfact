### 1.1 Endpoint Definitions

| Endpoint | Method | Purpose | AC | HTTP Codes |

| `/api/domains/{domainId}/data-sets` | POST | Create new dataset | AC1 | 201, 400, 404, 500 |
| `/api/domains/{domainId}/data-sets` | GET | List all domain's datasets | AC1 | 200, 404 |

### 1.2 Request/Response Flows

#### Flow 1: POST /api/domains/{domainId}/data-sets (Creation)

```
REQUEST:
POST /api/domains/123/data-sets
Content-Type: application/json

{
  "numberOfRows": 100,
  "datasetName": "Q1_2026_Customers",
  "columns": [
    {
      "name": "firstname",
      "columnType": "FIRST_NAME",
      "additionalConfig": null
    },
    {
      "name": "amount",
      "columnType": "AMOUNT",
      "additionalConfig": "{\"min\": 100, \"max\": 10000}"
    }
  ]
}

VALIDATION STEPS:
1. Path param: domainId > 0 and domain exists
2. Body: numberOfRows > 0, <= 10000
3. Body: columns not empty, all types valid
4. Body: datasetName not empty, <= 255 chars
5. Logical: Domain must be active (not soft-deleted)

PROCESSING:
1. Call DataGeneratorService.generate(request)
2. Receive GenerationResponseDTO (in-memory data)
3. Create DataSet entity (persist metadata + JSON data)
4. Return created DataSet metadata

RESPONSE (201 Created):
Location: /api/data-sets/456
Content-Type: application/json

{
  "success": true,
  "message": "Dataset generated and saved successfully",
  "data": {
    "id": 456,
    "domainId": 123,
    "name": "Q1_2026_Customers",

    "updatedAt": "2026-02-27T14:32:00"

}
```

#### DataSet Entity (Existing - Analyzed)

```java
@Entity
@Table(name = "datasets", 
       indexes = {@Index(name = "idx_domain_id", columnList = "domainId")})
public class DataSet {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private Long domainId;                 // Foreign key to Domain

    @Column(nullable = false, length = 255)
    private String name;                   // Dataset name

    @Column(nullable = false)
    private Integer rowCount;              // For reporting

    @Column(nullable = false)
    private Long generationTimeMs;         // Performance metric

    @Column(columnDefinition = "LONGTEXT") // For JSON serialization
    private String dataJson;               // Complete generated data

    @Column(nullable = false, updatable = false)
    private LocalDateTime createdAt;       // Auto-set on insert

    @Column(nullable = false)
    private LocalDateTime updatedAt;       // Auto-set on update

    @Column(nullable = true)
    private LocalDateTime deletedAt;       // Soft delete support

    @PrePersist @PreUpdate // Timestamp management
    protected void updateTimestamps() { ... }
}
```

**Rationale:**
- `dataJson` as LONGTEXT allows storing generated data (1000 rows × 10 cols ≈ 100KB easily fits)
- Index on `domainId` for fast lookup by domain
- Soft delete via `deletedAt` for audit trail
- JPA handles timestamp auto-updates

## 3. Controller Design & Validation Strategy

### 3.1 DataGenerationController Skeleton

```java
@RestController
@RequestMapping("/api")
public class DataGenerationController {
    
    @Autowired
    private DataGeneratorService dataGeneratorService;
    
    @Autowired
    private DataSetRepository dataSetRepository;
    
    @Autowired
    private DomainRepository domainRepository;
    
    // POST - Create dataset
    @PostMapping("/domains/{domainId}/data-sets")
    public ResponseEntity<ApiResponse<DataSetDTO>> generateDataset(
            @PathVariable Long domainId,
            @Valid @RequestBody GenerationRequestDTO request) {
        
        // Validation: Domain exists
        domainRepository.findByIdAndDeletedAtIsNull(domainId)
            .orElseThrow(() -> new EntityNotFoundException("Domain not found"));
        
        // Validation: Set domainId in request
        request.setDomainId(domainId);
        
        // Generate stream (core logic)
        GenerationResponseDTO response = dataGeneratorService.generate(request);
        
        // Persist to database
        DataSet dataset = createDataSetEntity(domainId, request, response);
        DataSet saved = dataSetRepository.save(dataset);
        
        // Return 201 Created (Location header auto-managed)
        return ResponseEntity
            .created(URI.create("/api/data-sets/" + saved.getId()))
            .body(ApiResponse.success(
                toDTO(saved),
                "Dataset generated and saved successfully"
            ));
    }
    
    // GET - List by domain
    @GetMapping("/domains/{domainId}/data-sets")
    public ResponseEntity<ApiResponse<List<DataSetDTO>>> listDatasetsByDomain(
            @PathVariable Long domainId) {
        
        // Validation: Domain exists
        domainRepository.findByIdAndDeletedAtIsNull(domainId)
            .orElseThrow(() -> new EntityNotFoundException("Domain not found"));
        
        // Query all datasets for domain
        List<DataSet> datasets = dataSetRepository
            .findByDomainIdAndDeletedAtIsNull(domainId);
        
        return ResponseEntity.ok(
            ApiResponse.success(
                datasets.stream().map(this::toDTO).collect(Collectors.toList()),
                "Retrieved " + datasets.size() + " dataset(s)"
            )
        );
    }
    
    // GET - Single metadata
    @GetMapping("/data-sets/{id}")
    public ResponseEntity<ApiResponse<DataSetDTO>> getDatasetMetadata(
            @PathVariable Long id) {
        
        DataSet dataset = dataSetRepository
            .findByIdAndDeletedAtIsNull(id)
            .orElseThrow(() -> new EntityNotFoundException("Dataset not found"));
        
        return ResponseEntity.ok(
            ApiResponse.success(toDTO(dataset), "Dataset metadata retrieved")
        );
    }
    
    // GET - Data with pagination
    @GetMapping("/data-sets/{id}/data")
    public ResponseEntity<ApiResponse<PaginatedDataResponseDTO>> getDatasetData(
            @PathVariable Long id,
            @RequestParam(defaultValue = "0") int page,
            @RequestParam(defaultValue = "50") int size) {
        
        // Validation: pagination params
        if (size < 1 || size > 100) {
            throw new IllegalArgumentException("Page size must be between 1 and 100");
        }
        if (page < 0) {
            throw new IllegalArgumentException("Page must be >= 0");
        }
        
        DataSet dataset = dataSetRepository
            .findByIdAndDeletedAtIsNull(id)
            .orElseThrow(() -> new EntityNotFoundException("Dataset not found"));
        
        // Parse JSON and apply pagination
        PaginatedDataResponseDTO paged = paginateData(dataset, page, size);
        
        return ResponseEntity.ok(
            ApiResponse.success(paged, "Data retrieved successfully")
        );
    }
    
    // DELETE - Soft delete
    @DeleteMapping("/data-sets/{id}")
    public ResponseEntity<Void> deleteDataset(@PathVariable Long id) {
        
        DataSet dataset = dataSetRepository
            .findByIdAndDeletedAtIsNull(id)
            .orElseThrow(() -> new EntityNotFoundException("Dataset not found"));
        
        // Soft delete via timestamp
        dataset.setDeletedAt(LocalDateTime.now());
        dataSetRepository.save(dataset);
        
        return ResponseEntity.noContent().build();
    }
    
    // Helper methods
    private DataSet createDataSetEntity(
            Long domainId, 
            GenerationRequestDTO request, 
            GenerationResponseDTO response) {
        
        DataSet dataset = new DataSet();
        dataset.setDomainId(domainId);
        dataset.setName(request.getDatasetName() != null ? 
            request.getDatasetName() : 
            "Dataset_" + System.currentTimeMillis());
        dataset.setRowCount(response.getNumberOfRows());
        dataset.setGenerationTimeMs(response.getGenerationTimeMs());
        dataset.setDataJson(serializeData(response.getData()));
        return dataset;
    }
    
    private DataSetDTO toDTO(DataSet entity) {
        DataSetDTO dto = new DataSetDTO();
        dto.setId(entity.getId());
        dto.setDomainId(entity.getDomainId());
        dto.setName(entity.getName());
        dto.setRowCount(entity.getRowCount());
        dto.setGenerationTimeMs(entity.getGenerationTimeMs());
        dto.setCreatedAt(entity.getCreatedAt());
        dto.setUpdatedAt(entity.getUpdatedAt());
        return dto;
    }
    
    private PaginatedDataResponseDTO paginateData(
            DataSet dataset, 
            int page, 
            int size) {
        List<Map<String, Object>> allData = deserializeData(dataset.getDataJson());
        int total = allData.size();
        int totalPages = (total + size - 1) / size;
        int start = page * size;
        int end = Math.min(start + size, total);
        
        return new PaginatedDataResponseDTO(
            total,
            page,
            size,
            totalPages,
            allData.subList(start, end)
        );
    }
}
```

### 3.2 Validation Strategy

| Level | Validation Rule | Location | Exception |
|-------|-----------------|----------|-----------|
| **Path Parameter** | domainId > 0 | Controller method | PathVariableException → 400 |
| **Domain Existence** | Domain found & not deleted | Repository query | EntityNotFoundException → 404 |
| **Request Body** | @Valid annotation | GenerationRequestDTO | MethodArgumentNotValidException → 400 |
| **Pagination** | size ∈ [1, 100], page ≥ 0 | Method body | IllegalArgumentException → 400 |
| **Dataset Existence** | Dataset found & not deleted | Repository query | EntityNotFoundException → 404 |
| **Data Integrity** | JSON parsing successful | Service helper | JsonProcessingException → 500 |

### 4.1 Error Response Format

```json

  "success": false,

  "data": null,
  "timestamp": "2026-02-27T14:32:00",
```

### 4.2 HTTP Status Codes

| Status | When | AC |
|--------|------|-----|
| 201 | Dataset created successfully | AC1 |
| 200 | GET successful (metadata, data, list) | AC2-AC3 |
| 204 | DELETE successful (no content) | AC4 |
| 400 | Invalid request (negative rows, invalid type, bad pagination) | AC1 |
| 404 | Domain/Dataset not found or soft-deleted | AC1-AC4 |
| 500 | Unexpected error (DB, JSON parsing) | Error handling |

### 4.3 Edge Cases

| Scenario | Handling | Example |
|----------|----------|---------|
| Large dataset (10K rows) | In-memory generation + JSON serialization | POST with numberOfRows=10000 |
| Empty columns list | Reject with 400 | columns: [] → validation error |
| Invalid column type | GeneratorFactory throws IllegalArgumentException | columnType: "INVALID" |
| Domain soft-deleted | Treat as 404 | GET /api/domains/111/data-sets where 111 deleted |
| Pagination: page=100, size=50 but only 50 rows | Return empty `rows` array | GET /data-sets/123/data?page=100&size=50 |
| Very large dataJson serialization | LONGTEXT column supports up to 4GB | 10K × 10 cols ≈ 1-5MB typical |

## 5. Transaction & Consistency

### 5.1 Transaction Boundaries

```
Transaction Context:
┌─── generateDataset() ─────────┐
│ 1. Validation (read-only)    │
│ 2. DataGeneratorService.gen  │  ← May be separate transaction
│ 3. DataSet entity creation   │
│ 4. dataSetRepository.save()  │  ← Single WRITE transaction
│ 5. Return response           │
└──────────────────────────────┘
```

**Key Points:**
- DataGeneratorService.generate() is stateless, no DB writes
- DataSet persistence is ACID-compliant (JPA/Spring transaction)
- Soft delete is atomic (single UPDATE query)
- No distributed transactions needed

### 5.2 Concurrency Considerations

- Database index on `domainId` prevents N+1 queries
- Immutable `createdAt` field (set in @PrePersist, never updated)
- Optimistic locking not required (no concurrent updates to same dataset)
- Soft delete pattern avoids foreign key cascades

## 6. Performance Considerations

### 6.1 Profiling Points

| Operation | Est. Time | Mitigation |
|-----------|-----------|-----------|
| Generate 100 rows × 10 cols | 50-100ms | In-memory, no DB calls |
| JSON serialization | 5-20ms | Single conversion at save |
| DB INSERT (DataSet) | 2-5ms | Indexed, single row |
| DB SELECT list (100 datasets) | 5-10ms | Index on domainId |
| Pagination (deserialize + slice) | 10-50ms | In-memory, depends on row count |

### 6.2 Optimization Strategy

1. **In-memory generation** (S2.1) - No DB latency during generation
2. **Single JSON blob storage** - Avoid normalizing data rows (10K inserts)
3. **Database indexing** - @Index on domainId for fast filtering
4. **Pagination limits** - Max 100 rows/page prevents memory overload
5. **Soft deletes** - No FK constraint cascades (fast delete)

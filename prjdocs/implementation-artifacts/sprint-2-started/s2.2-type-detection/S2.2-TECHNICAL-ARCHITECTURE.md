## 🏗️ Architecture Overview

### **High-Level Flow**

```
User uploads CSV file
    ↓
POST /api/domains/{id}/detect-types (S2.5 from frontend)
    ↓
TypeDetectionController (receives multipart CSV)
    ↓
TypeDetectionService (business logic)
    ↓
DetectorFactory (instantiate appropriate detectors)
    ↓
ColumnTypeDetector[] (13 specialized detectors run in parallel)
    ↓
Confidence scoring + Alternative suggestions
    ↓
TypeDetectionResult (response DTO)
    ↓
Return JSON with detected types + alternatives
```

## 🔌 Core Components

### **1. ColumnTypeDetector (Abstract Base Class)**

**Location:** `com.movkfact.service.detection.ColumnTypeDetector`

```java
/**
 * Abstract base class for detecting column data types.
 * 
 * Strategy pattern: Each concrete detector handles one ColumnType.
 * Parallel to S2.1's DataTypeGenerator pattern.
 */
public abstract class ColumnTypeDetector {
  
  // Configuration (can be overridden per subclass)
  protected List<String> headerPatterns;        // ["email", "mail", "e_mail"]
  protected List<Pattern> valueRegexPatterns;   // Compiled regex for validation
  protected int minimumConfidence = 80;         // Threshold for "confident" match
  protected double headerWeighting = 0.4;       // 40% weight to header
  protected double valueWeighting = 0.6;        // 60% weight to value analysis
  
  /**
   * Core detection method: Analyze column header + sample values
   * @return DetectionResult with type, confidence, alternatives
   */
  public final DetectionResult detect(String columnHeader, List<String> sampleValues) {
    int headerScore = matchHeaderPatterns(columnHeader);      // 0-100
    int valueScore = analyzeValuePatterns(sampleValues);      // 0-100
    
    // Weighted combination
    int combinedScore = Math.round(
      (headerScore * headerWeighting) + (valueScore * valueWeighting)
    );
    
    return new DetectionResult(
      getColumnType(),           // FIRST_NAME, EMAIL, etc.
      combinedScore,             // 0-100 confidence
      headerScore,
      valueScore,
      getAlternativeSuggestions(columnHeader, sampleValues)  // Top 3 candidates
    );
  }
  
  /**
   * Match header name against known patterns
   * @return 0-100 score (0 = no match, 100 = perfect match)
   */
  protected abstract int matchHeaderPatterns(String header);
  
  /**
   * Analyze sample values against expected patterns
   * @return 0-100 score
   */
  protected abstract int analyzeValuePatterns(List<String> sampleValues);
  
  /**
   * Get this detector's column type (e.g., EMAIL, FIRST_NAME)
   */
  protected abstract ColumnType getColumnType();
  
  /**
   * Suggest alternative types if primary is uncertain
   */
  protected abstract List<TypeAlternative> getAlternativeSuggestions(
    String columnHeader, List<String> sampleValues
  );
  
  // ========== HELPER METHODS ==========
  
  /**
   * Count valid values matching pattern (skip nulls)
   */
  protected int countValidMatches(List<String> values, Pattern regex) {
    return (int) values.stream()
      .filter(v -> v != null && !v.trim().isEmpty())
      .filter(v -> regex.matcher(v).find())
      .count();
  }
  
  /**
   * Calculate confidence = validMatches / totalNonNullValues
   */
  protected int calculateConfidence(List<String> values, Pattern regex) {
    long nonNullCount = values.stream()
      .filter(v -> v != null && !v.trim().isEmpty())
      .count();
    
    if (nonNullCount == 0) return 0; // All nulls
    
    int matches = countValidMatches(values, regex);
    return (int) ((double) matches / nonNullCount * 100);
  }
  
  /**
   * Normalize header (remove underscores, convert to lowercase)
   */
  protected String normalizeHeader(String header) {
    if (header == null) return "";
    return header.toLowerCase()
      .replaceAll("[_-]", "")
      .trim();
  }
  
  /**
   * Check if header contains any of the pattern keywords
   */
  protected boolean headerContainsPattern(String header, List<String> patterns) {
    String normalized = normalizeHeader(header);
    return patterns.stream().anyMatch(normalized::contains);
  }
}
```

### **2. Example Detector: EmailTypeDetector**

```java
public class EmailTypeDetector extends ColumnTypeDetector {
  
  private static final Pattern EMAIL_REGEX = 
    Pattern.compile("^[A-Za-z0-9+_.-]+@[A-Za-z0-9.-]+\\.[A-Z|a-z]{2,}$");
  
  public EmailTypeDetector() {
    this.headerPatterns = List.of(
      "email", "mail", "e-mail", "e_mail", "emailaddress", 
      "contact_email", "email_address"
    );
    this.valueRegexPatterns = List.of(EMAIL_REGEX);
    this.minimumConfidence = 80;
  }
  
  @Override
  protected int matchHeaderPatterns(String header) {
    String normalized = normalizeHeader(header);
    
    if (headerContainsPattern(header, headerPatterns)) {
      // Exact match (e.g., "email" or "email_address")
      if (normalized.equals("email") || 
          normalized.equals("emailaddress") ||
          normalized.equals("email_address")) {
        return 98;  // Very high confidence
      }
      
      // Contains one of the patterns
      return 85;
    }
    
    return 0;  // No match
  }
  
  @Override
  protected int analyzeValuePatterns(List<String> sampleValues) {
    if (sampleValues == null || sampleValues.isEmpty()) {
      return 0;
    }
    
    int matches = countValidMatches(sampleValues, EMAIL_REGEX);
    int totalNonNull = (int) sampleValues.stream()
      .filter(v -> v != null && !v.trim().isEmpty())
      .count();
    
    if (totalNonNull == 0) return 0;
    
    return (int) ((double) matches / totalNonNull * 100);
  }
  
  @Override
  protected ColumnType getColumnType() {
    return ColumnType.EMAIL;
  }
  
  @Override
  protected List<TypeAlternative> getAlternativeSuggestions(
      String columnHeader, List<String> sampleValues) {
    // Emails rarely confused with other types, return empty
    return List.of();
  }
}
```

### **3. DetectorFactory**

**Location:** `com.movkfact.service.detection.DetectorFactory`

```java
/**
 * Factory for instantiating appropriate ColumnTypeDetectors.
 * 
 * Strategy pattern factory (similar to S2.1's GeneratorFactory).
 */
public class DetectorFactory {
  
  private static final List<Class<? extends ColumnTypeDetector>> DETECTOR_CLASSES = 
    List.of(
      // Personal detectors (6)
      FirstNameTypeDetector.class,
      LastNameTypeDetector.class,
      EmailTypeDetector.class,
      PhoneTypeDetector.class,
      GenderTypeDetector.class,
      AddressTypeDetector.class,
      
      // Financial detectors (3)
      AccountNumberTypeDetector.class,
      AmountTypeDetector.class,
      CurrencyTypeDetector.class,
      
      // Temporal detectors (4)
      DateBirthTypeDetector.class,
      DateCreatedTypeDetector.class,
      TimezoneTypeDetector.class,
      TimeTypeDetector.class,
      
      // Generic fallback
      GenericIdentifierDetector.class
    );
  
  /**
   * Create all detectors (to be run in parallel)
   */
  public static List<ColumnTypeDetector> createAllDetectors() {
    return DETECTOR_CLASSES.stream()
      .map(clazz -> {
        try {
          return clazz.getDeclaredConstructor().newInstance();
        } catch (Exception e) {
          throw new RuntimeException("Failed to instantiate detector: " + clazz, e);
        }
      })
      .collect(Collectors.toList());
  }
  
  /**
   * Run all detectors in parallel for a single column
   * Returns single result: primary type + alternatives
   */
  public static DetectionResult detectColumn(
      String columnHeader, List<String> sampleValues) {
    
    List<ColumnTypeDetector> detectors = createAllDetectors();
    
    // Run all detectors in parallel
    List<DetectionResult> results = detectors.parallelStream()
      .map(detector -> detector.detect(columnHeader, sampleValues))
      .collect(Collectors.toList());
    
    // Sort by confidence (descending)
    results.sort((r1, r2) -> Integer.compare(r2.getConfidence(), r1.getConfidence()));
    
    // Return top result + top 2 alternatives
    DetectionResult primary = results.get(0);
    List<DetectionResult> topAlternatives = results.stream()
      .skip(1)
      .limit(2)
      .filter(r -> r.getConfidence() > 20)  // Only show if non-trivial
      .collect(Collectors.toList());
    
    return primary.withAlternatives(topAlternatives);
  }
}
```

## 🔧 Technical Decisions

### **Decision 1: Parallel vs. Sequential Execution**

**Question:** Should detectors run in parallel or sequence?

**Options:**
- A) Sequential: Run 13 detectors one by one
  - Simpler, easier to debug
  - Slower: ~13x detector execution time
  
- B) Parallel: Run all 13 simultaneously (parallelStream)
  - Faster: ~2x detector execution time (constant overhead)
  - Thread-safe (no mutable state)

**Decision:** ✅ **Parallel execution**
- Detectors are stateless, no shared mutable state
- CompletableFuture or parallelStream() both OK
- Recommended: parallelStream() for simplicity

---

### **Decision 2: Header vs. Value Weighting**

**Question:** How much do we trust header names vs. values?

**Options:**
- 50/50: Equal weight
- 60/40: Trust values more
- 40/60: Trust headers more (headers are explicit)

**Decision:** ✅ **40% headers, 60% values**
- Headers can be misleading ("name", "date" are ambiguous)
- Values are more reliable indicators
- Formula: `confidence = 0.4 * headerScore + 0.6 * valueScore`

---

### **Decision 3: Confidence Threshold for "Detected"**

**Question:** When is a detection "confident enough"?

**Options:**
- 80%+: Very confident
- 70%+: Moderately confident
- 50%+: Weakly confident

**Decision:** ✅ **80% = CONFIRMED, 50-80% = UNCERTAIN**
- 80%+: Show as primary suggestion (user rarely needs to change)
- 50-80%: Show with alternatives (user should review)
- <50%: Return UNKNOWN (user must choose from all options)

---

### **Decision 4: How Many Detectors?**

**Question:** One detector per ColumnType or per category?

**Options:**
- A) 13 detectors: One for each ColumnType (FIRST_NAME, LAST_NAME, EMAIL, etc.)
  - Granular, easier to test individually
  - Simpler logic per detector
  
- B) 3 detectors: Personal, Financial, Temporal
  - Simpler factory
  - More complex per detector (handles multiple types)

**Decision:** ✅ **13 detectors (one per ColumnType)**
- Aligns with S2.1 ColumnType enum
- Each detector has clear responsibility
- Easier to add new types in future

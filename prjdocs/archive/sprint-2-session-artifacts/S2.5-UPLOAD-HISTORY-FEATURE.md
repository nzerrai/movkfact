# S2.5 Enhancement: CSV Upload History Display

**Status**: ✅ COMPLETE  
**Date**: 1 Mars 2026  
**Feature**: Display uploaded datasets list after CSV upload

---

## Problem Statement

Utilisateur rapporté: "Le test de l'IHM ne montre pas le résultat d'un upload CSV, je ne vois pas la liste des CSV uploadés"

**Root Cause**: Le composant `CsvUploadPanel` affichait uniquement le résumé de l'upload courant, mais ne persistait pas ni n'affichait l'historique des uploads précédents.

---

## Solution Delivered

### 1. New Component: **UploadedDatasetsList**

**File**: `src/components/CsvUploadPanel/UploadedDatasetsList.jsx` (300+ LOC)

**Purpose**: Display list of uploaded datasets for a domain via API

**Features**:
- ✅ Loads datasets from backend API: `GET /api/domains/{domainId}/data-sets`
- ✅ Displays datasets in table with:
  - Dataset name & file size
  - Row and column count
  - Status (Active/Failed)
  - Creation timestamp
- ✅ Action buttons:
  - View dataset → navigate to data viewer
  - Delete dataset → with confirmation
  - Refresh list → reload from API
- ✅ Loading/error/empty states
- ✅ File size formatting (B, KB, MB, GB)
- ✅ Date formatting (locale-aware)

**Key Methods**:
```jsx
// Load datasets from API
const loadDatasets = async () => { ... }

// Handle dataset deletion
const handleDelete = async (datasetId) => { ... }

// Format utilities
const formatFileSize = (bytes) => { ... }
const formatDate = (dateString) => { ... }
```

### 2. Enhanced Component: **CsvUploadPanel**

**File**: `src/components/CsvUploadPanel/CsvUploadPanel.jsx` (changes)

**Changes**:
```diff
+ import UploadedDatasetsList from './UploadedDatasetsList';

+ const [refreshTrigger, setRefreshTrigger] = useState(0);

  // In "confirmed" step:
  <Box sx={{ display: 'flex', gap: 1, justifyContent: 'space-between', mb: 3 }}>
    <Button onClick={() => setStep('review')}>← Back</Button>
    <Button onClick={() => {
+     // Trigger refresh of datasets list
+     setRefreshTrigger(prev => prev + 1);
      onProceedToConfiguration(detectionResults);
    }}>
      Proceed to Configuration →
    </Button>
  </Box>

+ {/* Display uploaded datasets list */}
+ {domainId && (
+   <UploadedDatasetsList
+     key={refreshTrigger}
+     domainId={domainId}
+     onViewDataset={(dataset) => console.log('View:', dataset)}
+     onDeleteDataset={(datasetId) => setRefreshTrigger(prev => prev + 1)}
+     showActions={true}
+   />
+ )}
```

**Result**: After uploading and confirming a CSV, users now see:
1. Upload summary (rows, columns, detection success)
2. Detected columns table
3. **NEW**: List of all uploaded datasets for the domain

## Usage Example

### In Application Code
```jsx
import CsvUploadPanel from './components/CsvUploadPanel/CsvUploadPanel';

function ConfigurationPage() {
  return (
    <CsvUploadPanel
      domainId={1}
      onProceedToConfiguration={(detectionResults) => {
        // Navigate to configuration page with results
      }}
      onCancel={() => {
        // Go back
      }}
    />
  );
}
```

### Component Props
```jsx
CsvUploadPanel:
- domainId: number (required) - Domain to upload to
- onProceedToConfiguration: fn(detectionResults) - Called on confirm
- onCancel: fn() - Called on cancel

UploadedDatasetsList:
- domainId: number (required) - Domain to load datasets from
- onViewDataset: fn(dataset) - Callback when View clicked
- onDeleteDataset: fn(datasetId) - Callback when Delete clicked
- showActions: boolean (default: true) - Show action buttons
```

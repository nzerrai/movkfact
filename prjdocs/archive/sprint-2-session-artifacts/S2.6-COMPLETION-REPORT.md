## Feature Implementation

### 1. PersonalFieldConfig Component
**Purpose**: Type-specific configuration form for Personal data types

**Supported Types** (6 total):
- `FIRST_NAME` - Locale selector (FR/US/DE/ES)
- `LAST_NAME` - Locale selector
- `EMAIL` - Domain pattern input
- `GENDER` - Format dropdown (Full/Short/Codes/Français)
- `PHONE` - Country code selector (+33, +1, +49, +34, +44)
- `ADDRESS` - Country selector

**Code Pattern**:
```jsx
<Switch> {
  case 'FIRST_NAME':
    return <FormControl with Locale Select>;
  case 'EMAIL':
    return <TextField for domain>;
  // ... other types
}
```

### 2. FinancialFieldConfig Component
**Purpose**: Type-specific configuration for Financial data types

**Supported Types** (3 total):
- `AMOUNT` - Currency dropdown + min/max amount ranges (1-999,999)
- `ACCOUNT_NUMBER` - Format selector (IBAN/BBAN/Masked/Generic)
- `CURRENCY` - Currency code dropdown (EUR/USD/GBP/JPY)

**Features**:
- Auto-populate currency code when selected
- Numeric input validation for min < max
- Range sliders for amount boundaries

### 3. TemporalFieldConfig Component
**Purpose**: Type-specific configuration for Temporal data types

**Supported Types** (4 total):
- `DATE_BIRTH` - Year range inputs (birth year range: 1900-current year)
- `DATE` - Date range with format selector (ISO/EU/US/Timestamp)
- `TIME` - Format selector (24H/12H/Milliseconds)
- `TIMEZONE` - IANA timezone dropdown

**Date Format Options**:
- ISO: YYYY-MM-DD
- EU: DD/MM/YYYY
- US: MM/DD/YYYY  
- Timestamp: Unix epoch

### 4. ConfigurationPanel Component (Main Orchestrator)
**Purpose**: Central hub coordinating all configuration forms, validation, and API calls

**Key Methods**:
- `handleConfigChange()` - Updates columnConfigs[column] state
- `validateInputs()` - Validates rowCount (1-10,000), CSV data exists
- `handleGenerate()` - POST to S2.3 API endpoint with payload:
  ```json
  {
    "numberOfRows": 1000,
    "columns": [
      {
        "name": "firstName",
        "type": "FIRST_NAME",
        "config": { "locale": "FR" }
      },
      ...
    ]
  }
  ```
- `setStep('results')` - Switches to ResultViewer display

**Flow**:
1. CsvUploadPanel (S2.5) → ConfigurationPanel step 1 (configure)
2. User fills forms (type-specific configs)
3. Click "Generate" button
4. Validate + call S2.3 API
5. Switch to ResultViewer step 2 (results display)

**State Management**:
- `columnConfigs` - object with per-column configurations
- `rowCount` - integer 1-10,000
- `isGenerating` - boolean loading state
- `generationResult` - dataset response from S2.3
- `step` - current UI tab ('configure' | 'results')

### 5. ResultViewer Component
**Purpose**: Display generated data with export and navigation options

**Export Capabilities**:
- **JSON Export**: Uses FileSaver.js to download formatted JSON
- **CSV Export**: Calls S2.4 API `GET /api/data-sets/{id}/export/download`
- **Clipboard**: Navigator clipboard API for JSON copy

**Display Features**:
- Statistics card (row count, generation time, column count)
- Data preview table (first 10 rows, scrollable)
- JSON preview with syntax highlighting (react-json-view)
- Error handling with Alert snackbars

**Navigation Buttons**:
- "Configure More Data" - Back to ConfigurationPanel
- "Finish & Back to Home" - Exit to home page

## API Integrations

### S2.3 API (Data Generation)
**Endpoint**: `POST /api/domains/{domainId}/data-sets`

**Request**:
```json
{
  "numberOfRows": 1000,
  "columns": [
    {
      "name": "columnName",
      "type": "COLUMN_TYPE",
      "config": { "type-specific": "parameters" }
    }
  ]
}
```

**Response** (successful):
```json
{
  "data": {
    "id": 1,
    "domainId": 1,
    "rowCount": 1000,
    "generationTimeMs": 125,
    "data": [...],
    "createdAt": "2026-03-01T08:45:00Z"
  }
}
```

**Integration in ConfigurationPanel**:
- Called on "Generate" button click
- Payload built from columnConfigs + rowCount
- Response triggers ResultViewer display

### S2.4 API (CSV Export)
**Endpoint**: `GET /api/data-sets/{datasetId}/export/download`

**Parameters**:
- `format` (optional): 'csv' | 'json' (defaults to original format)
- `pretty` (optional): true | false for JSON formatting

**Response**:
- Binary blob (file download)
- Content-Disposition header with filename
- MIME type: text/csv

**Integration in ResultViewer**:
- Called on "Download CSV" button
- Uses async/await with loading state
- FileSaver.js handles download

## Design Decisions Applied

### Row Count Limits
**Decision**: Maximum 10,000 rows for MVP
**Rationale**: Prevent browser memory exhaustion with large datasets
**Implementation**: `rowCount` validation in ConfigurationPanel

### CSV Export Strategy
**Decision**: Use S2.4 backend API instead of frontend conversion
**Rationale**: Consistency with backend serialization, better performance
**Implementation**: `handleDownloadCsv()` calls `/api/data-sets/{id}/export/download`

### Component Organization
**Decision**: Separate type-specific configs into sub-components
**Rationale**: Maintainability, testability, reusability for future types
**Implementation**: PersonalFieldConfig, FinancialFieldConfig, TemporalFieldConfig imported by ConfigurationPanel

### Data Flow
**Decision**: React Context (CsvContext) → ConfigurationPanel → S2.3 API → ResultViewer → S2.4 API
**Rationale**: Unidirectional data flow, single source of truth
**Implementation**: Props drilling with callbacks for form updates

## 🎯 Core Algorithm (Simplified)

```
For EACH CSV column:
  1. Score header name match (patterns like "email", "first_name")     → 0-100%
  2. Score sample values match (regex patterns, format validation)    → 0-100%
  3. Combine scores: confidence = 0.4×header + 0.6×values             → Final %
  4. Rank all 13 detectors by confidence
  5. Return: Primary type + Top 2 alternatives (if confidence < 80%)
```

**Key Formula:** `confidence = (header_score × 0.4) + (value_score × 0.6)`
- **Why?** Headers are explicit but often ambiguous ("name", "date")
- **Why?** Values are ground truth (actual data reliability)
- **Weighting:** 40/60 balances both signals

## 📋 13 Column Type Detectors

### **Personal Data (6)**
- FIRST_NAME (patterns: "firstname", "first_name", etc.)
- LAST_NAME (patterns: "lastname", "last_name", etc.)
- EMAIL (pattern: `^[A-Za-z0-9+_.-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}$`)
- PHONE (pattern: international format with country codes)
- GENDER (values: M/F or Male/Female)
- ADDRESS (patterns: street patterns, postal codes)

### **Financial (3)**
- ACCOUNT_NUMBER (patterns: IBAN, account IDs)
- AMOUNT (pattern: decimal numbers, currency symbols)
- CURRENCY (values: EUR, USD, GBP, etc.)

### **Temporal (4)**
- DATE_BIRTH (pattern: YYYY-MM-DD, with "birth" context)
- DATE_CREATED (pattern: ISO date with "created" context)
- TIMEZONE (values: IANA timezone strings)
- TIME (pattern: HH:MM or HH:MM:SS)

## ⚙️ Technical Approach

### **Architecture: Strategy + Factory Pattern**

```
ColumnTypeDetector (abstract base)
  ├── EmailTypeDetector (matches "email" header + email regex)
  ├── FirstNameTypeDetector (matches "first_name" header + name patterns)
  ├── AmountTypeDetector (matches "amount" header + decimal patterns)
  └── ... (13 detectors total)

DetectorFactory
  ├── createAllDetectors() → List of 13 detectors
  ├── detectColumn(header, values) → Parallel execution
  └── Result: DetectionResult (primary + 2 alternatives)
```

### **Parallel Execution**
```java
results = detectors.parallelStream()
  .map(detector → detector.detect(columnHeader, sampleValues))
  .collect()  // All 13 run simultaneously
  
results.sort(confidence descending)
return results.get(0) + alternatives(top 2)
```

### **Performance: Sampling Strategy**
- File < 100 rows: Read all
- File > 100 rows: Sample first 50 + every 10th + random 50 = ~195 rows
- Result: 10K rows analyzed in ~50ms instead of 500ms

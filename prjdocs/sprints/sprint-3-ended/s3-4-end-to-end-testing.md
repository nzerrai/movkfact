# Story 3.4: End-to-End Testing & Test Automation (COMPREHENSIVE)

Status: ready-for-dev

## Story

As a QA engineer (Quinn),
I want to implement a complete E2E test suite covering all user-facing features (domain management, CSV upload, data generation, batch jobs, WebSocket notifications),
so that regressions are caught automatically and the team has confidence in the full system before S3.5 Docker deployment.

## Acceptance Criteria

**AC1 — E2E Framework Setup**
- Cypress 13 installed and configured in `movkfact-frontend/`
- `cypress.config.js` created at frontend root
- Scripts added to `package.json`: `test:e2e` (interactive), `test:e2e:headless` (CI)
- Tests execute against backend on `http://localhost:8080` and frontend on `http://localhost:3000`

**AC2 — Core Happy Path (Scenario 1)**
- Test: Create domain → Upload CSV → Configure generation → Generate → Download JSON
- Execution time < 5 seconds end-to-end
- Asserts: success toast visible, JSON download file exists

**AC3 — Domain Management CRUD (Scenario 2)**
- Test: Create / Read / Update / Delete domain operations
- All operations reflected correctly in the UI

**AC4 — CSV Upload Validation (Scenario 3)**
- 4 sub-scenarios: valid CSV, missing-column CSV, large CSV (10k+ rows), invalid format (non-CSV)
- Error messages displayed for invalid inputs; generation button disabled when validation fails

**AC5 — Data Generation Configuration (Scenario 4)**
- Test: column selection, count configuration, type selection
- Asserts: only selected columns in output, correct row count

**AC6 — Batch Generation (Scenario 5)**
- Test: submit batch with 3 datasets; verify jobId returned (202), status RUNNING then COMPLETED
- Execution time < 10 seconds for 3000 total rows

**AC7 — WebSocket Real-Time Notifications (Scenario 6)**
- Test: verify STOMP messages received on `/topic/batch/{jobId}` (job_started → job_progress_update → job_completed)
- Messages arrive within 500ms of actual progress; timestamps increase monotonically

**AC8 — Error Handling & Recovery (Scenario 7)**
- 4 sub-scenarios: dataset failure mid-batch, WebSocket disconnection + reconnect, retry exhaustion, invalid domainId (400)
- Partial failure → final status PARTIAL_FAILURE (some datasets succeeded)

**AC9 — Activity Tracking History (Scenario 8)**
- Test: complete workflow and verify activity log shows CREATED / VIEWED / DOWNLOADED actions
- Reset endpoint produces RESET action in activity history

**AC10 — Performance Benchmarks (Scenario 9)**
- Domain creation: < 100ms
- CSV upload parsing: < 500ms (10k rows)
- Single dataset generation (1000 rows): < 2 seconds
- Batch generation (10 × 1000 rows): < 10 seconds
- WebSocket message latency: < 500ms

**AC11 — UI Responsiveness & Accessibility (Scenario 10)**
- UI remains interactive during long batch operations (NotificationPanel updates live)
- Tab navigation works through all controls
- Form labels associated with inputs; error messages linked to fields
- Color contrast meets WCAG AA

**AC12 — Backend Coverage >= 80%**

**AC13 — Frontend Coverage >= 80%**

## Dev Notes

### Environment

  - Start: `./mvnw spring-boot:run -Dspring-boot.run.profiles=dev` from project root

### API Endpoints (complete, all public for MVP)

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/health` | Health check |
| POST | `/api/domains` | Create domain `{"name": "..."}` |
| GET | `/api/domains` | List all domains |
| GET | `/api/domains/{id}` | Get domain |
| PUT | `/api/domains/{id}` | Update domain |
| DELETE | `/api/domains/{id}` | Delete domain |
| POST | `/api/domains/{domainId}/data-sets` | Generate single dataset |
| GET | `/api/domains/{domainId}/data-sets` | List domain datasets |
| GET | `/api/data-sets/{id}` | Get dataset metadata |
| GET | `/api/data-sets/{id}/data?page=0&size=100` | Paginated data |
| DELETE | `/api/data-sets/{id}` | Soft delete dataset |
| POST | `/api/batch/generate` | Submit batch job (202 + jobId) |
| GET | `/api/batch/{jobId}` | Job status |
| GET | `/api/batch/{jobId}/progress` | Detailed progress DTO |
| GET | `/api/batch/{jobId}/full-status` | Full persistent status |
| GET | `/api/domains/{domainId}/activities` | Activity history |

**Single dataset request:**
```json
{
  "datasetName": "TestDataset",
  "count": 100,
  "columns": [
    { "name": "first_name", "type": "FIRST_NAME" },
    { "name": "email", "type": "EMAIL" }
  ]
}
```

**Batch request:**
```json
{
  "datasets": [
    { "domainId": 1, "datasetName": "DS-A", "count": 1000,
      "columns": [{ "name": "first_name", "type": "FIRST_NAME" }] },
    { "domainId": 1, "datasetName": "DS-B", "count": 500,
      "columns": [{ "name": "email", "type": "EMAIL" }] }
  ]
}
```

**Batch response:** `202 Accepted`, body `{"jobId": 42, "status": "RUNNING"}` — jobId is a **Long**.

**Activity response:** list of `{id, actionType, timestamp, datasetId, datasetName}`. ActionTypes: `CREATED, VIEWED, DOWNLOADED, MODIFIED, RESET`.

### WebSocket Protocol

- Endpoint: `http://localhost:8080/ws/batch-notifications` (SockJS transport)
- STOMP topic per job: `/topic/batch/{jobId}` — jobId as String in topic path
- Message types: `job_started`, `job_progress_update`, `job_completed`, `job_error`
- Frontend service: `movkfact-frontend/src/services/WebSocketService.js`
  - `WebSocketService.subscribeToBatch(jobIdStr, callback)` — jobId **must be String**
  - `WebSocketService.unsubscribeFromBatch(jobIdStr)`
- Context: `BatchJobsContext` (`movkfact-frontend/src/context/BatchJobsContext.jsx`) — tracks active jobs, persists RUNNING jobs to `localStorage` key `movkfact_active_jobs`

**Cypress pattern for WebSocket testing** (poll API — SockJS is not natively testable in Cypress):
```javascript
// cypress/support/commands.js
Cypress.Commands.add('waitForJobCompletion', (jobId) => {
  const poll = (retries = 30) => {
    return cy.request(`http://localhost:8080/api/batch/${jobId}`)
      .then(({ body }) => {
        if (body.status === 'COMPLETED' || body.status === 'FAILED') return body;
        if (retries <= 0) throw new Error('Job timed out');
        return cy.wait(500).then(() => poll(retries - 1));
      });
  };
  return poll();
});
```

For full STOMP message verification (Scenario 6), use a `cy.task` with a Node-side SockJS client, or verify indirectly via `GET /api/batch/{jobId}/full-status` fields.

### CI/CD Pipeline Template

**`.github/workflows/test.yml`:**
```yaml
name: Test Suite
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  backend-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'
          cache: 'maven'
      - run: mvn clean test jacoco:report
      - uses: actions/upload-artifact@v4
        with:
          name: jacoco-report
          path: target/site/jacoco/

  frontend-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'
          cache-dependency-path: movkfact-frontend/package-lock.json
      - run: cd movkfact-frontend && npm ci
      - run: cd movkfact-frontend && npm run test:coverage
      - uses: actions/upload-artifact@v4
        with:
          name: frontend-coverage
          path: movkfact-frontend/coverage/

  e2e-tests:
    runs-on: ubuntu-latest
    needs: [backend-tests, frontend-tests]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'
      - uses: actions/setup-node@v4
        with:
          node-version: '18'
      - name: Start backend (background)
        run: mvn spring-boot:run -Dspring-boot.run.profiles=dev &
      - name: Wait for backend
        run: |
          for i in {1..30}; do
            curl -s http://localhost:8080/api/health && break
            sleep 2
          done
      - run: cd movkfact-frontend && npm ci && npm run build
      - name: Start frontend (background)
        run: npx serve -s movkfact-frontend/build -p 3000 &
      - run: cd movkfact-frontend && npm run test:e2e:headless
      - uses: actions/upload-artifact@v4
        if: failure()
        with:
          name: cypress-videos
          path: movkfact-frontend/cypress/videos/
```

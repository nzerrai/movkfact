## Problem Summary

After integrating `ConfigurationPanel` into the `DomainsPage` workflow, users encountered an error when attempting to generate datasets via the UI:

```
ColumnConfigDTO and ColumnType must not be null
```

### Root Cause

The `ConfigurationPanel` component was sending configuration data to the backend with the wrong JSON field names.

**Frontend was sending:**
```json
{
  "numberOfRows": 1000,
  "columns": [
    {
      "name": "firstName",
      "type": "FIRST_NAME",      // ❌ Wrong key
      "config": { "locale": "FR" }  // ❌ Wrong structure
    }
  ]
}
```

**Backend expected (ColumnConfigDTO):**
```java
public class ColumnConfigDTO {
    private String name;              // ✅ Required
    private ColumnType columnType;    // ✅ NOT "type"!
    private String format;            // Optional
    private Integer minValue;         // Optional
    private Integer maxValue;         // Optional
    private Boolean nullable;         // Optional
    private String additionalConfig;  // Optional
}
```

### Why It Failed

1. JSON field `type` doesn't map to Java field `columnType`
2. Jackson deserialization set `columnType` to `null`
3. `GeneratorFactory.createGenerator()` validated: `if (columnConfig == null || columnConfig.getColumnType() == null)`
4. Error thrown: "ColumnConfigDTO and ColumnType must not be null"

---

## Solution Implemented

Modified `ConfigurationPanel.handleGenerate()` to correctly map frontend data to backend DTO structure.

### Changes Made

**File:** `/home/seplos/mockfact/movkfact-frontend/src/components/DataConfigurationPanel/ConfigurationPanel.jsx`

**Before:**
```javascript
const columns = Object.entries(columnConfigs).map(([colName, config]) => ({
  name: colName,
  type: config.type,              // ❌ Wrong key
  config: config.params           // ❌ Wrong structure
}));
```

**After:**
```javascript
const columns = Object.entries(columnConfigs).map(([colName, config]) => {
  const colDto = {
    name: colName,
    columnType: config.type  // ✅ Correct key: columnType
  };
  
  const params = config.params || {};
  
  // Map optional parameters to DTO fields
  if (params.format) colDto.format = params.format;
  if (params.minValue !== undefined) colDto.minValue = params.minValue;
  if (params.maxValue !== undefined) colDto.maxValue = params.maxValue;
  if (params.nullable !== undefined) colDto.nullable = params.nullable;
  
  // Store type-specific parameters in additionalConfig
  if (Object.keys(params).length > 0) {
    colDto.additionalConfig = JSON.stringify(params);
  }
  
  return colDto;
});
```

### Key Fixes

1. **Field Mapping:** `type` → `columnType` (matches Java DTO)
2. **Structure:** Flat fields instead of nested `config` object
3. **Optional Parameters:** Map to individual DTO fields (`format`, `minValue`, `maxValue`, `nullable`)
4. **Additional Config:** Store type-specific parameters as JSON string in `additionalConfig`
